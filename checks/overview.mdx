title: Built-in checks
description: Overview of Spectre's server-side anticheat detection system

Spectre includes several built-in checks that detect common movement exploits and malicious client behavior. Each check runs on the server during movement processing and assigns violation levels (VL) when suspicious activity is detected.

## How checks work

Checks are executed during the `onMovementUpdate` lifecycle hook, which processes every player movement packet. When a check detects suspicious behavior, it returns a `CheckResult` with:

- **Failure reason**: A descriptive string identifying what was detected
- **VL weight**: The violation level weight to add to the player's total

Violation levels accumulate over time. Thresholds and VL weights are hardcoded in the source but can be modified by editing the check implementations.

## Understanding VL weights

Each check assigns a different VL weight based on severity:

- **Flight**: 10.0 (instant ban threshold)
- **Mantle**: 5.0 (severe exploit)
- **Relative speed**: 3.0 (moderate)
- **Packet rate**: 3.0 (moderate)
- **Speed**: 2.0 (minor, may be lag)

<Info>
VL weights are cumulative. A player who triggers multiple speed violations will eventually reach your configured punishment threshold.
</Info>

## Customizing checks

You can disable individual checks or create your own by extending the `Check` class:

```java
public class MyCustomCheck extends Check {
    @Override
    public String getName() {
        return "MyCheck";
    }

    @Override
    public String getDescription() {
        return "Detects custom behavior";
    }

    @Override
    public CheckResult onMovementUpdate(
        UUID playerId,
        PlayerRef playerRef,
        PlayerInput.InputUpdate update,
        Vector3d fromPosition,
        Vector3d toPosition,
        MovementStates movementStates,
        MovementManager movementManager,
        float dt,
        boolean inServerVelocity
    ) {
        // Your detection logic
        if (suspicious) {
            return CheckResult.fail("Custom Violation", 5.0);
        }
        return CheckResult.passed();
    }
}
```

Since Spectre doesn't expose a public API, you'll need to modify the source to register custom checks. Add your check registration in `ACInputSystem.java`:

```java
private void registerChecks() {
    checkManager.registerCheck(new SpeedCheck());
    checkManager.registerCheck(new RelativeSpeedCheck());
    checkManager.registerCheck(new FlightCheck());
    checkManager.registerCheck(new MantleCheck());
    checkManager.registerCheck(new PacketRateCheck());
    checkManager.registerCheck(new MyCustomCheck()); // Add your check here
}
```
