Welcome to the Spectre contributing guide. We're building a server-side anticheat framework for Hytale, and your contributions make a real difference in keeping servers fair and exploiters out.

## How you can help

Spectre is an open-source project, and we welcome all kinds of contributions. Whether you're writing code, improving documentation, or reporting issues, everything helps.

### Code contributions

You can contribute to Spectre by:

- **Building new checks** — Implement detection for movement exploits, combat cheats, or other violations
- **Fixing bugs** — Resolve false positives or issues in existing checks
- **Improving performance** — Optimize check logic to reduce server overhead
- **Refactoring** — Clean up code to make it more maintainable

### Documentation contributions

Good documentation makes Spectre accessible to everyone:

- Improve setup instructions or troubleshooting guides
- Document check behavior and configuration options
- Add examples and use cases
- Fix typos or clarify confusing sections

### Reporting issues

You don't need to write code to help. Reporting bugs and suggesting features is equally valuable:

- **Bug reports** — If you encounter a false positive or something isn't working right, [open a bug report](https://github.com/yourusername/spectre/issues/new?template=bug_report.md)
- **Feature requests** — Have an idea for a new check or improvement? [Submit a feature request](https://github.com/yourusername/spectre/issues/new?template=feature_request.md)

<Note>
When reporting false positives, include detailed reproduction steps and server logs. This helps us diagnose and fix the issue faster.
</Note>

## What to work on

If you're looking for something to contribute:

1. Check the [issue tracker](https://github.com/yourusername/spectre/issues) for open bugs and feature requests
2. Look for issues labeled `good first issue` if you're new to the project
3. Browse the codebase in `src/main/java/dev/solarion/anticheat/` to understand how checks are structured

## Getting started

Ready to contribute? Here's what to do next:

<Steps>
  <Step title="Set up your development environment">
    Follow the [development setup guide](/contributing/development-setup) to install dependencies and build Spectre locally.
  </Step>
  
  <Step title="Review the contribution guidelines">
    Read our [contribution guidelines](/contributing/guidelines) to understand code style, testing requirements, and the PR process.
  </Step>
  
  <Step title="Make your changes">
    Create a branch, implement your changes, and test them on a local Hytale server.
  </Step>
  
  <Step title="Submit a pull request">
    Open a PR with a clear description of what you've changed and why.
  </Step>
</Steps>

## Questions?

If you're unsure about something or need help getting started, don't hesitate to ask. Open a discussion or comment on an existing issue — we're here to help.

Thank you for contributing to Spectre. Your work helps keep Hytale servers fair for everyone.
